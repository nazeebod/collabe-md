# Стартовый экран

## Описание изменения

Реализована главная страница сервиса — точка входа по маршруту `/`. Пользователь видит описание сервиса и может одним нажатием создать новый Markdown-документ. После создания выполняется переход на страницу документа `/d/:id`.

## Структура данных

### Таблица `documents` (PostgreSQL)

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | UUID | Первичный ключ, генерируется автоматически |
| `title` | text | Заголовок документа, по умолчанию «Untitled» |
| `yjs_state` | bytea | Сериализованное состояние Yjs (заполняется при сохранении) |
| `created_at` | timestamptz | Дата создания |
| `updated_at` | timestamptz | Дата последнего обновления |

### API-контракты (`packages/shared`)

```typescript
type CreateDocumentResponse = {
  id: string;
  url: string;  // "/d/{id}"
};

type ApiResponse<T> = {
  success: boolean;
  data: T | null;
  error: string | null;
};
```

## Интерфейс

### Главная страница (`/`)

- Бейдж «Real-time collaboration».
- Заголовок «Collabe MD».
- Описание: создание документа, совместное редактирование, доступ по ссылке без аккаунтов.
- Кнопка **Create document** (`data-testid="create-document"`).
  - В состоянии загрузки: текст «Creating...», кнопка disabled.
  - При ошибке: текст ошибки под кнопкой.

## Бизнес-логика и валидация

1. Нажатие **Create document** → `POST /api/documents` с телом `{ title?: string }`.
2. Если `title` не передан, сервер сохраняет «Untitled».
3. При успехе клиент получает `{ id, url }` и выполняет `navigate(url)`.
4. Документ добавляется в список недавних (см. change «недавние документы»).
5. Rate limit: 100 запросов в минуту на API.

## Права доступа (RBAC)

Аутентификация отсутствует. Создание документа доступно любому посетителю.

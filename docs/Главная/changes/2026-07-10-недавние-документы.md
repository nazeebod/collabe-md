# Недавние документы

## Описание изменения

На главной странице отображается список недавно открытых или созданных документов. Данные хранятся локально в `localStorage` браузера — без серверной синхронизации. Пользователь может быстро вернуться к документу или убрать его из списка.

## Структура данных

### LocalStorage

Ключ: `collabe-md-recent-documents`

```typescript
type RecentDocument = {
  id: string;       // UUID документа
  title: string;    // актуальный заголовок на момент добавления
  visitedAt: string; // ISO 8601, время последнего открытия/создания
};
```

Массив, максимум **20** записей. При добавлении дубликат по `id` удаляется, новая запись вставляется в начало.

## Интерфейс

### Блок «Your recent documents»

- Заголовок и подпись «Stored only in this browser».
- Список пунктов (`data-testid="recent-documents"`):
  - Ссылка на `/d/{id}` с заголовком и датой открытия.
  - Кнопка «×» для удаления из списка (`aria-label` с названием документа).
- Если список пуст — блок не рендерится.

### Формат даты

`visitedAt` отображается через `toLocaleString`: месяц (кратко), день, часы и минуты. При невалидной дате — «Recently opened».

## Бизнес-логика и валидация

1. **Добавление в список** (`addRecentDocument`):
   - При создании документа на главной (`HomePage`).
   - При открытии документа (`DocumentPage` — загрузка метаданных).
   - При сохранении заголовка (`DocumentPage` — blur поля title).
2. **Удаление** (`removeRecentDocument`): только из localStorage, документ на сервере не затрагивается.
3. **Чтение** (`getRecentDocuments`): сортировка по `visitedAt` убыванию.
4. **Валидация при чтении**: некорректный JSON или невалидные элементы — ключ очищается, возвращается пустой массив.
5. **Обновление UI**: `HomePage` передаёт `refreshKey` в `RecentDocuments` для перерисовки после создания.

## Права доступа (RBAC)

Не применимо — данные локальные, без серверного доступа.

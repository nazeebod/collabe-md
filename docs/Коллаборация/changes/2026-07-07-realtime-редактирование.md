# Realtime-редактирование

## Описание изменения

Реализована совместная работа над текстом документа в реальном времени через Yjs CRDT и WebSocket-сервер Hocuspocus. Участники видят присутствие друг друга и цветные курсоры в редакторе. Состояние документа персистится в PostgreSQL.

## Структура данных

### AwarenessUser (`packages/shared`)

```typescript
type AwarenessUser = {
  name: string;       // случайное имя животного
  color: string;      // hex, цвет курсора
  colorLight: string; // hex + alpha "50", фон курсора
};
```

20 имён животных, 10 цветов — выбираются случайно при подключении (`createAwarenessUser`).

### Yjs

- `Y.Doc` на клиенте, комната = `documentId`.
- `Y.Text("content")` — CRDT-текст Markdown.
- Awareness state: `{ user: AwarenessUser }` через HocuspocusProvider.

### WebSocket

| Окружение | URL |
|-----------|-----|
| Development | `ws://localhost:3002` |
| Production | `ws(s)://{host}/collaboration` |

### Персистенция (PostgreSQL)

- `onLoadDocument`: чтение `yjs_state` → `Y.applyUpdate`.
- `onStoreDocument`: `Y.encodeStateAsUpdate` → запись в `yjs_state` (debounce 2s, max 10s).
- `onConnect`: проверка существования документа (`documentExists`), иначе отказ в подключении.

## Интерфейс

### PresenceBar

- Метка «Online:».
- Список плашек с именами (`data-testid="presence-bar"`, `presence-user`).
- Цвет фона плашки = `user.color`.
- Обновление по событию `awareness.on("change")`.

### Курсоры в редакторе

- `yCollab(ytext, provider.awareness, { undoManager })` в CodeMirror 6.
- Удалённые курсоры с именем и цветом участника.

### Статус соединения

- `connecting` → «Loading editor...».
- `disconnected` → баннер «Connection lost. Reconnecting...».
- `connected` → скрытый `data-testid="connection-status"`.

## Бизнес-логика и валидация

1. `useCollaboration(documentId)` создаёт `Y.Doc`, `HocuspocusProvider`, подписывается на `ytext.observe`.
2. Local awareness: `provider.awareness.setLocalStateField("user", awarenessUser)`.
3. При `onSynced` / `onConnect` / `onStatus` обновляется `ConnectionStatus`.
4. Cleanup при размонтировании: `provider.destroy()`, `ydoc.destroy()`.
5. E2E-тест: два браузера, общий текст в preview, 2 presence-user на каждой странице.

## Права доступа (RBAC)

Подключение к WS разрешено только для существующих документов (UUID в БД). Редактирование — любой с валидной ссылкой.

# Плашка GitHub

## Описание изменения

На всех страницах сервиса добавлена фиксированная плашка-ссылка на репозиторий проекта на GitHub. Плашка отображается поверх контента в правом верхнем углу и ведёт на исходный код в новой вкладке.

## Структура данных

Изменений в серверной структуре данных нет.

### Конфигурация (клиент)

```typescript
// apps/web/src/lib/config.ts
export const GITHUB_REPO_URL =
  import.meta.env.VITE_GITHUB_REPO_URL ?? "https://github.com/muroed/collabemd";
```

Переопределяется через переменную окружения `VITE_GITHUB_REPO_URL` при сборке.

## Интерфейс

### GitHubBanner

- Фиксированная позиция, правый верхний угол.
- Ссылка `<a>` с иконкой GitHub и текстом «GitHub».
- `target="_blank"`, `rel="noopener noreferrer"`.
- `aria-label="View source on GitHub"`.
- `data-testid="github-banner"`.

Плашка рендерится в `App.tsx` вне `<Routes>` — видна и на главной, и на странице документа.

## Бизнес-логика и валидация

- URL по умолчанию: `https://github.com/muroed/collabemd`.
- При деплое можно указать другой репозиторий через `VITE_GITHUB_REPO_URL`.
- Плашка не влияет на маршрутизацию и бизнес-логику документов.

## Права доступа (RBAC)

Не применимо — статическая внешняя ссылка.
